تصميم قاعدة البيانات و ERD لنظام موازنات وفوركاست SaaS

تصميم مبدئي احترافي قابل للتطوير لنظام يخدم المصانع وشركات التجزئة الغذائية وسلاسل الفروع، مع دعم تعدد الشركات، الموازنات، الفوركاست، الفعلي، السيناريوهات، واعتمادات الإدارة.

1) الفكرة المعمارية للنظام

النظام مبني كمنصة SaaS متعددة الشركات، كل عميل يمتلك شركة أو أكثر داخل النظام. كل شركة يمكن أن تحتوي على مصانع، فروع، مخازن، منتجات، خامات، مراكز تكلفة، ومستخدمين بصلاحيات مختلفة. التصميم يسمح بربط البيانات المالية والتشغيلية في نموذج واحد للموازنة والفوركاست.

Multi-Tenant
كل شركة ببيانات مستقلة.
Budget Cycle
موازنة سنوية/ربع سنوية/شهرية.
Forecast Engine
توقعات متجددة حسب الفعلي.
Approval Workflow
اعتمادات ومراجعات وسجل تدقيق.

2) ERD العام للنظام

ملاحظة: الرسم يعرض العلاقات الأساسية. التنفيذ النهائي يمكن تقسيمه إلى Modules داخل Laravel/PHP أو أي Framework مناسب، مع إضافة Indexes وViews للتقارير السريعة.

5) التصميم التفصيلي لدورة الموازنة والفوركاست

  1. إنشاء Budget Cycle للسنة المالية.
  2. إدخال أو استيراد افتراضات المبيعات، الإنتاج، الشراء، المصروفات، والرواتب.
  3. إضافة Budget Lines لكل شهر وحساب وفرع ومنتج.
  4. إرسال الموازنة للاعتماد حسب الصلاحيات.
  5. استيراد Actuals شهرياً من Excel أو ERP أو POS.
  6. توليد Forecast Cycle بناءً على الفعلي + الأشهر المتبقية.
  7. حساب Budget vs Actual vs Forecast.
  8. إصدار تقارير الإدارة: P&L، Cash Flow، Margin، Branch Profitability.

7) قواعد مهمة في تصميم SaaS

  • كل جدول تشغيلي يجب أن يحتوي على tenant_id أو يصل إليه عبر company_id لضمان فصل بيانات العملاء.
  • استخدام soft delete للجداول الحساسة مثل المنتجات والموازنات.
  • حفظ الإصدارات للموازنة والفوركاست وعدم تعديل النسخة المعتمدة مباشرة.
  • تفعيل Audit Log لكل تعديل مالي.
  • إضافة Import Templates لكل نوع بيانات لتسهيل إدخال العملاء.
  • دعم العملات والفترات المالية المختلفة.
  • تخزين الافتراضات المعقدة في JSON مع الاحتفاظ بالبيانات الأساسية في أعمدة قابلة للفلترة.
